Skeleton provides a framework for design systems in frontend web applications. Built on a foundation of [Tailwind CSS](https://tailwindcss.com/), we provide a robust theme system and vast suite of components. While also establishing a simple set of patterns and conventions for you and your team to follow. We solve the hard problems of building design systems at scale, so you can ship faster.

## What is Skeleton?

### Design System

With Skeleton, the design system isn't an afterthought, it's the core pillar. Providing themes, colors, typography, spacing, and more. With opinionated guardrails for applying your application's design.

### Tailwind Components

A library of reusable CSS utility classes that enable you to quickly style native HTML elements such as buttons, forms, tables, and more.

### Framework Components

These go beyond presentation, and enable state and logic. Built atop battle-tested primitives via [Zag.js](https://zagjs.com/), from the creators of [Chakra UI](https://chakra-ui.com/). While fully adapting to your design system out of the box.

## Our Philosophy

> Placeholder: Graphic Here

### Framework Agnostic Core

Skeleton core requires only Tailwind CSS. Beyond that, you're free to use your favorite framework of choice. Including simple bolt-on components popular frameworks.

### Native-First

We aim to embrace the interface of the web, not replace it. Skeleton defaults to the use of semantic HTML elements and native browser APIs. Beyond ease of use, this offers the best story for accessibility.

### Accessibility

Accessibility is paramount and serves as a core pillar for how we design and implement every part of Skeleton.

### Utility-First

Skeleton is opt-in by default, following the [utility-first methodology](https://tailwindcss.com/docs/styling-with-utility-classes) established by Tailwind CSS, while extending the capabilities in meaningful ways.

### Adaptive Design

Skeleton is designed to adapt to your project's aesthetic, while still providing sane defaults out of the box.
